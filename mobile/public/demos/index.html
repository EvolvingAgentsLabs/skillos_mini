LLM-OS validation demos

Two games running on the same kernel as llm_os. Both download a 230 MB local model on first load, then play entirely in the browser via WebAssembly. Used for kernel validation: Tetris exercises a single-loop arcade pattern; Scavenger exercises a multi-step quest with the same compiled-state shape that skillos_robot's SceneGraph emits.

▶ Tetris

A 350M-parameter LLM plays Tetris using grammar-constrained ISA opcodes. Phase-controlled state machine: observe → moves → halt on game over.

cartridge: game/tetris · 4 methods · 5 enumerable move actions

▶ Scavenger

A 350M-parameter LLM solves a multi-step quest. Find the red cube, pick it up, deliver to the blue square — through walls, around pits, with a 30-move budget. The compiled state shape mirrors skillos_robot's SceneGraph: labeled objects with bearings and distances, ready for direct port to a real-world physical scavenger task.

cartridge: game/scavenger · 4 methods · 7 enumerable opcodes

▶ Kernel-mode runner (PR 3 validation)

End-to-end test of the kernel running a generic multi-turn agentic loop — not tied to a specific game. Pick a cartridge (echo, tetris), enter a goal, and watch grammar-enforced ISA opcodes flow turn by turn. Stub tool handlers acknowledge each call; the kernel + sampler do the real work. This is the testbed for the runGoal kernel-mode integration.

module: src/lib/kernel/runner.ts · cartridges: echo, tetris
Validation purpose. These demos prove the llm_os kernel runs unmodified inside skillos_mini's static-asset bundle, with two different cartridges, on the same model and the same wllama runtime the rest of the app uses. They are not production cartridges — production cartridges live in cartridge-v2/cartridges/ and run through the v2 cartridge runner. Once kernel-mode lands in runGoal (skillos_mini PR 3), v2 cartridges will run on this same kernel via the v2_adapter.

Headers. SharedArrayBuffer requires Cross-Origin-Opener-Policy: same-origin + Cross-Origin-Embedder-Policy: require-corp. Vite's dev server includes a small plugin to send these (see vite.config.ts). The Capacitor production build includes them via the server config.
